# scenario.json 写 JSON 时的转义速查

## 核心原则

`description` 和 `info`/`on_failure` 字段是 agent 念给玩家听的叙事文本，里面可能含：
- 双引号 `"` — NPC 对话
- 反斜杠 `\` — 路径
- 换行 `\n` — 多行文本

## 转义表

| 字符 | JSON 里写 | 示例 |
|------|----------|------|
| `"` | `\"` | `"他说：\"别去。\""` |
| `\` | `\\` | `"路径：C:\\docs"` |
| 换行 | `\n` | `"第一行\n第二行"` |

## ❌ 常见翻车

```json
// 翻车：内层引号没转义
"description": "他低声说："你不能进去。"——门锁了。"
```

## ✅ 正确写法

```json
// 用单引号替代双引号（推荐，可读性最好）
"description": "他低声说：'你不能进去。'——门锁了。"

// 或用反斜杠转义
"description": "他低声说：\"你不能进去。\"——门锁了。"

// 中文场景：用全角引号「」替代 ASCII 引号（消灭转义地狱）
"description": "笔记中记录了名为「最受人尊敬的圣剑克鲁」的团体。——这是盖文最后想去的地方。"
```

`「」` 是中文全角引号（U+300C/U+300D），在 JSON 字符串中无需转义。对于含多处引用的中文描述文本，用「」比逐个 `\"` 转义更干净。英文对话仍优先用单引号。

## 写完立即 validate

```bash
python3 scripts/game.py validate
```

JSON 语法错误会被 Python 直接报出具体行号。
