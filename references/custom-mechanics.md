# 自定义机制追踪

部分模组引入了标准 COC 7 版以外的自定义追踪值（相信值、腐化值、进度条等）。这些值需要在跨会话中持久化，但不适合放在 characters.json 的标准字段中。

## 追踪方式

### 方案 A：通过 campaign-memory.md 手动记录

最简单的方式——在 `campaign-memory.md` 中维护一个追踪区块：

```markdown
## 自定义追踪
- 相信值: 5
  - +3: 购买三色手绳 (Day1 船库)
  - +2: 萨麦迪降临 SAN 损失 (Day1 晚宴)
```

每次变更时更新。跨会话通过 resume 的 campaign_notes 自动注入。

### 方案 B：通过 state.json 追踪

在 state.json 的顶层级增加自定义字段：

```bash
# 初始化
python3 -c "
import json
with open('state.json') as f: s = json.load(f)
s['custom_values'] = {'相信值': 0}
with open('state.json','w') as f: json.dump(s, f, indent=2, ensure_ascii=False)
"
```

### 方案 C：在 characters.json 的 notes 字段中嵌入

在角色 notes 中追加结构化追踪：

```json
"notes": "... | 相信值:5 (+3手绳 +2晚宴SAN)"
```

每次变更时 parse → 修改 → 写回。

**推荐方案 A**（最简单，不会破坏现有命令的 schema）。

## 相信值机制（胖绅士模块实例）

- **定义**：每次 SAN 损失 → 相信值增加相应数值
- **效果**：相信值 >5 → 看到更清晰的恐怖幻象。相信值 ≥10 → 成为祭品资格者
- **追踪**：agent 在每次 SAN 损失后手动更新 campaign-memory.md 中的追踪值
- **叙事体现**：在描述恐怖现象时，根据相信值决定描述的清晰度和细节程度

### 三色手绳与相信值的互动

手绳（红黑白三色）是真品——伊莱贾编的伏都教防护道具。关键互动规则：
- **佩戴期间**：手绳提供防护，但每次防护都会加深玩家对手绳的依赖（相信值+3 初始佩戴）
- **高相信值状态下**（>10）：玩家能通过手绳看到灵魂层面的东西——自己的金色灵魂、他人的灵魂状态（马特奥的烛火、威廉的空壳）
- **法阵牵引时**：手绳可作为「反向锚点」——当玩家的相信方向与法阵预期不一致时（如愤怒的信 vs 恐惧的信），手绳会反向拉回灵魂而非协助抽取
- **「不纯的信」判定**：邪教法阵喂养的是恐惧驱动的盲信（被吓到信、求着信、跪着信）。当玩家以愤怒、反抗、被耍后的暴怒等方式产生相信时，法阵无法正常消化这种「信」，出现波动或反噬

## agent 职责

1. 在第一次触发自定义机制时，向玩家简述规则（不剧透具体阈值）
2. 每次变更后记录到 campaign-memory.md
3. 根据当前值调整叙事细节（低值=模糊、高值=清晰恐怖）
4. 不要频繁向玩家汇报当前值（会打破沉浸），只在玩家主动问或值跨阈值时提醒
